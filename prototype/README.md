# Prototype
あらかじめ用意しておいた「原型」からインスタンスを生成するようにするためのパターン

### メリット
* 原型となるオブジェクトを生成するのにコストがかかったり、複雑な場合、隠蔽できる
* オブジェクトの型を知らなくても新しいオブジェクトを取得できる

### デメリット
* オブジェクトのコピーを生成することが複雑だったりする

- inodeインターフェースにもclone機能がある
- fileとfolder構造体はどちらもタイプ
